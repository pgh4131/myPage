# PRD: 개인 자기소개 사이트

## 1. 프로젝트 개요

### 1.1 프로젝트명

개인 자기소개 및 방명록 웹사이트

### 1.2 프로젝트 목적

사용자의 기본 프로필, 관심사, 기술 스택, 학습 방향성을 한눈에 보여줄 수 있는 자기소개 사이트를 제작한다. 사이트는 단순한 정보 나열이 아니라, 사용자의 성장 과정과 개발자로서의 방향성을 자연스럽게 드러내는 것을 목표로 한다.

### 1.3 프로젝트 성격

* 깔끔하고 모던한 개발자 포트폴리오 느낌
* 과하게 발랄하거나 장식적인 분위기는 지양
* 동기 친구들이 편하게 둘러보고 사용자를 이해할 수 있는 구조
* 자기소개와 가벼운 상호작용이 공존하는 개인 사이트

---

## 2. 목표

### 2.1 핵심 목표

* 방문자가 사이트에 들어왔을 때 사용자가 누구인지 빠르게 이해할 수 있어야 한다
* 현재 관심사와 기술 스택, 앞으로의 학습 방향을 자연스럽게 전달해야 한다
* 사이트만의 특색 요소로 성장 타임라인을 제공해야 한다
* 방명록 기능을 통해 방문자와 가볍게 상호작용할 수 있어야 한다

### 2.2 성공 기준

* 첫 화면에서 이름, 학과, 한 줄 소개가 명확히 보인다
* 방문자가 스크롤만으로 주요 정보를 모두 확인할 수 있다
* 성장 타임라인이 사이트의 차별화 요소로 기능한다
* 익명 방명록 작성이 문제없이 가능하다
* 데스크톱과 모바일에서 모두 읽기 쉽고 자연스럽게 동작한다

---

## 3. 타겟 사용자

### 3.1 주요 방문자

* 동기 친구들
* 가까운 지인
* 사용자의 관심사나 개발 방향이 궁금한 사람들

### 3.2 사용자 니즈

* 이 사람이 어떤 사람인지 빠르게 알고 싶다
* 무엇에 관심이 있는지 보고 싶다
* 어떤 기술을 사용할 수 있고, 무엇을 배우려 하는지 알고 싶다
* 간단한 인사나 응원 메시지를 남기고 싶다

## 5. 핵심 기능

## 5.1 프로필 정보 표시

사용자의 기본 정체성을 가장 먼저 보여주는 섹션이다.

### 포함 항목

* 이름: `{사용자 이름}`
* 학과: `{학과명}`
* 한 줄 소개: `{짧은 자기소개}`

### 목적

* 방문자가 첫 화면에서 사용자를 즉시 이해할 수 있어야 한다
* 사이트 전체 분위기를 대표하는 핵심 정보로 기능해야 한다

### 요구사항

* 메인 상단 Hero 영역에 배치
* 이름과 한 줄 소개는 시각적으로 강조
* 학과 정보는 보조 정보로 자연스럽게 노출
* 프로필 이미지 또는 아바타는 선택적으로 포함 가능

---

## 5.2 관심사 및 취미 소개

사용자의 성향과 개성을 보여주는 섹션이다.

### 포함 항목

* 현재 관심사
* 취미
* 자주 탐색하는 주제
* 개인적으로 좋아하는 활동

### 목적

* 단순한 이력 소개가 아니라 사람 자체에 대한 인상을 전달한다
* 방문자가 사용자에게 친근감을 느낄 수 있도록 한다

### 요구사항

* 카드형 또는 태그형 UI로 표현
* 텍스트가 길어지지 않도록 간결하게 정리
* 너무 많은 항목보다는 핵심 항목 위주로 배치

---

## 5.3 현재 기술 스택

현재 사용할 수 있는 기술을 정리해 보여주는 섹션이다.

### 포함 항목

* 프론트엔드 기술
* 언어
* 도구 및 플랫폼
* 기타 활용 가능 기술

### 목적

* 사용자의 현재 기술적 기반을 명확하게 보여준다
* 개발자 포트폴리오 느낌을 강화한다

### 요구사항

* 기술들을 카테고리별 또는 태그형으로 정리
* 가독성을 우선하며, 복잡한 숙련도 시각화는 MVP 범위에서 제외 가능
* 대표 기술이 한눈에 들어오도록 배치

---

## 5.4 앞으로 배워나갈 분야

앞으로 관심 있게 학습할 분야를 정리하는 섹션이다.

### 포함 항목

* 현재 배우는 중인 기술
* 앞으로 배우고 싶은 분야
* 확장하고 싶은 개발 역량

### 목적

* 사용자가 정적인 상태가 아니라 성장 중인 사람이라는 인상을 준다
* 단순 소개를 넘어 방향성과 의지를 전달한다

### 요구사항

* 현재 기술 스택 섹션과 구분되되 흐름은 자연스럽게 연결
* “배우는 중”, “앞으로 배우고 싶은 것”처럼 구분 가능
* 짧은 설명 문구 추가 가능

---

## 5.5 특색 기능: 성장 타임라인

사이트의 차별화 요소로, 사용자의 변화와 성장을 시간 흐름에 따라 보여주는 기능이다.

### 기능 설명

사용자의 프로젝트 경험, 학습 내용, 관심사 변화 등을 시간순으로 기록해 보여준다.

### 목적

* 사이트에 스토리성을 부여한다
* 단순한 프로필 페이지를 넘어, 성장형 자기소개 사이트로 보이게 한다
* 개발자 포트폴리오 감성을 강화한다

### 예시 항목

* `2026.03` 자기소개 사이트 기획 시작
* `2026.04` 새로운 프론트엔드 기술 학습 시작
* `2026.05` 첫 개인 기능 구현 완료

### 요구사항

* 날짜, 제목, 짧은 설명으로 구성
* 세로형 타임라인 UI 사용
* 모바일에서도 읽기 쉬운 구조 지원
* 최신순 또는 시간순 정렬 중 하나로 통일
* 초기 MVP에서는 3~5개 항목 정도만 제공해도 충분

---

## 5.6 방명록 페이지

방문자가 익명으로 메시지를 남길 수 있는 별도 페이지다.

### 기능 설명

방문자는 이름 대신 닉네임 또는 익명 상태로 짧은 메시지를 작성할 수 있다. 작성된 메시지는 목록 형태로 페이지에 표시된다.

### 입력 항목

* 닉네임 또는 익명 이름
* 메시지 내용

### 목적

* 동기 친구들이 가볍게 흔적을 남길 수 있도록 한다
* 사이트에 정적인 소개 이상의 상호작용 요소를 부여한다

### 요구사항

* 익명 작성 허용
* 작성 후 메시지 목록에 반영
* 최신 메시지가 상단에 보이도록 정렬
* 빈 입력값 제출 불가
* 메시지 길이 제한 필요
* 과도하게 긴 문장 또는 스팸성 입력 방지 필요

### 권장 제한

* 닉네임: 최대 20자
* 메시지: 최대 200자

## 7. 사용자 시나리오

### 시나리오 1

방문자는 메인 페이지에 접속해 이름, 학과, 한 줄 소개를 확인한다. 이후 스크롤을 내리며 관심사, 취미, 기술 스택을 보고 사용자가 어떤 사람인지 파악한다.

### 시나리오 2

방문자는 성장 타임라인을 보며 사용자가 최근 무엇을 배우고 있고 어떤 방향으로 성장하고 있는지 확인한다. 이를 통해 단순한 소개 페이지 이상이라는 인상을 받는다.

### 시나리오 3

방문자는 방명록 페이지로 이동해 익명으로 짧은 인사나 응원 메시지를 남긴다. 이후 다른 사람들이 남긴 메시지도 함께 읽을 수 있다.

## 9. 디자인 방향

### 9.1 디자인 키워드

* 깔끔함
* 모던함
* 개발자 포트폴리오 느낌
* 정돈된 정보 구조
* 과하지 않은 인터랙션

### 9.2 시각적 방향

* 흰색 또는 어두운 중성 배경 기반
* 포인트 컬러는 1개 또는 2개 이내 사용
* 카드형 레이아웃과 충분한 여백 활용
* 타이포그래피 중심의 정제된 디자인
* 불필요한 장식, 과한 캐릭터성, 과도한 애니메이션은 배제

### 9.3 UX 방향

* 메인 페이지는 위에서 아래로 자연스럽게 읽히는 흐름
* 정보는 섹션 단위로 명확히 분리
* 방명록 이동은 눈에 잘 띄지만 전체 흐름을 해치지 않도록 배치
* 인터랙션은 부드럽고 절제된 수준으로 구성

## 15. 최종 요약

본 프로젝트는 동기 친구들을 주요 방문자로 하는 개인 자기소개 사이트이며, 메인 1페이지 스크롤형 구조와 별도 방명록 페이지로 구성된다. 사이트는 깔끔하고 모던한 개발자 포트폴리오 느낌을 지향하며, 프로필 정보, 관심사, 기술 스택, 학습 방향성과 함께 성장 타임라인을 핵심 차별화 요소로 포함한다. 방명록은 익명 작성이 가능해야 하며, 전체 구현은 Next.js, Tailwind CSS, Supabase, Vercel 스택을 기준으로 진행한다.
